## thermo

[thermo](thermo/README.md)

Home thermostat automation over IR: Pi/Pico/ESP32 zones talk to a DMZ rendezvous service and drive Daikin head units. Active state in [`BOOKMARK.md`](thermo/BOOKMARK.md); backlog in [`TODO.md`](thermo/TODO.md).

### thermo/config

[thermo/config](thermo/config/README.md)

Committed deploy env files (`*.env`) and samples (`*.env.sample`) for zones, OAuth, SSH host keys, and tests. Select with `THERMO_ENV_FILE`; secrets live under `thermo/priv/`.

### thermo/consumed

[thermo/consumed](thermo/consumed/README.md)

Archived and superseded thermo planning notes (older DMZ and Pi deploy write-ups). Kept for history; current design lives in sibling trees.

### thermo/debuglogs

[thermo/debuglogs](thermo/debuglogs/README.md)

Local-only scratch space for Pi, DMZ, and container log dumps. UTC timestamp naming; nothing committed except this README and `.gitignore`.

### thermo/dmz

[thermo/dmz](thermo/dmz/README.md)

Internet-facing Flask rendezvous: zones POST state and receive commands; the controller posts commands and reads zone state. Dockerized Alpine runtime with no long-lived credentials.

### thermo/onboard

[thermo/onboard](thermo/onboard/README.md)

Zone firmware and deploy: shared Python in `common/`, hardware backends (Pi Zero 2 W, Pico2W, ESP32-S3), per-room `zones/`, and `install/` dispatch. Host tests via `make -C thermo/onboard test`.

### thermo/priv

thermo/priv

Gitignored secrets and private env overrides (zone keys, OAuth client secret, SSH host keys, per-device `.env`). Never commit contents here.

### thermo/scribble

thermo/scribble

Ad hoc IR capture, decode, and Daikin protocol experiments. Captures under `captures/`; scripts for bench work before changes land in onboard profiles.

### thermo/test

[thermo/test](thermo/test/README.md)

Black-box and integration tests that wire thermo microservices together. Host pytest and Docker-backed suites; see README for planned e2e coverage.

### thermo/ui

thermo/ui

Minimal HTML UI that proxies onboard or DMZ Flask APIs (`/ui/context`, `/ui/command`) for manual control and debugging.
